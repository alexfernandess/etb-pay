Classes Abstratas vs. Interfaces

Como a Banca Cobra

"Uma classe abstrata pode ser instanciada diretamente através do operador new, desde que todos os seus métodos abstratos sejam implementados no construtor." (FALSO - Nenhuma classe abstrata pode ser instanciada diretamente).

Como o Mercado Usa

Usamos interfaces para garantir desacoplamento (ex: padrão DAO). Uma classe abstrata Conta serve como molde genérico, mas instanciamos apenas ContaCorrente ou ContaPoupanca.

JDBC & Segurança (SQL Injection)

Como a Banca Cobra

As bancas da Cesgranrio costumam afirmar que a interface Statement é mais rápida e segura que a PreparedStatement para inserir dados de usuários. (FALSO - Statement é vulnerável a SQL Injection).

Como o Mercado Usa

No ETB-PAY, toda comunicação com o SQLite passa por PreparedStatement para sanitizar a entrada do usuário (senhas, CPFs) via placeholders (?).

Questão 01

Cesgranrio - Caixa (Adaptada 2024)

Na API JDBC do Java, para executar uma instrução SQL que contém parâmetros de entrada dinâmicos fornecidos pelo usuário, de modo a prevenir ataques de SQL Injection e otimizar a pré-compilação da consulta no SGBD, o desenvolvedor deve utilizar a interface:

  1. Statement
  2. CallableStatement
  3. ResultSet
  4. PreparedStatement
  5. Connection

Gabarito: D

Comentário da Banca: A interface PreparedStatement pré-compila a instrução SQL e parametriza as entradas usando ?, neutralizando tentativas de injetar comandos SQL maliciosos.

Dica Estratégica: Cesgranrio sempre atrela Statement a vulnerabilidade de segurança e PreparedStatement a performance/segurança.

Questão 02

FGV - TSE Unificado (Adaptada 2024)

Sobre os conceitos de Orientação a Objetos no Java a partir da versão 8, assinale a alternativa correta em relação ao uso de Interfaces:

  1. Interfaces não podem conter nenhum método com corpo implementado.
  2. Uma classe em Java pode implementar múltiplas interfaces, mitigando a restrição de herança múltipla de classes.
  3. Atributos declarados em uma interface são nativamente protected.
  4. Interfaces podem ser instanciadas diretamente utilizando o operador new Interface().
  5. Métodos em interfaces são implicitamente final, impedindo a sobrescrita.

Gabarito: B

Armadilha Semântica: A alternativa A era verdade no passado, mas desde o Java 8, interfaces podem ter métodos implementados usando a palavra-chave default. A banca da FGV adora testar quem estuda por materiais defasados.

Dica Estratégica: Herança múltipla de *estado* (classes) é proibida em Java, mas herança múltipla de *comportamento* (interfaces) é permitida.

Questão 03

CEBRASPE - BACEN (Adaptada 2024)

Considere o seguinte trecho de código Java onde ContaPoupanca herda de Conta e sobrescreve o método sacar().

Conta c = new ContaPoupanca(); c.sacar(100.0);

Durante a execução, devido ao princípio do Polimorfismo e da Ligação Dinâmica (Dynamic Binding), o método executado será o da classe Conta, pois o tipo da variável de referência determina a execução em tempo de compilação.

  1. Certo
  2. Errado

Gabarito: Errado

Comentário da Banca: O CEBRASPE inverteu o conceito conceitual. Na ligação dinâmica do Java, o método executado é determinado pelo tipo do objeto instanciado em memória (new ContaPoupanca()), e não pelo tipo da variável de referência (Conta c).

Dica Estratégica: Polimorfismo de sobreposição (Overriding) resolve-se em tempo de execução (Runtime). Lembre-se: O lado direito do sinal de igual (=) dita o comportamento real.

Questão 04

FCC - TRT (Adaptada 2024)

Na construção de um sistema corporativo utilizando a biblioteca Swing para a criação de uma interface gráfica, o programador decide utilizar classes abstratas para modelar a hierarquia de componentes. Assinale a característica primordial de uma classe abstract em Java:

  1. Pode ser instanciada livremente, desde que não contenha métodos abstratos.
  2. Obriga todas as suas subclasses a implementar todos os seus métodos concretos.
  3. Não pode ser instanciada diretamente através do operador new, servindo apenas como superclasse.
  4. Impede a declaração de atributos de instância, permitindo apenas constantes estáticas.
  5. Suporta a palavra-chave final na sua declaração, protegendo o código de herança acidental.

Gabarito: C

Comentário da Banca: O conceito basilar de uma classe abstrata é que ela é "incompleta" conceitualmente. Logo, a JVM bloqueia sua instanciação direta. A alternativa E é uma armadilha comum, pois abstract e final são modificadores mutuamente exclusivos (antagônicos).

Questão 05

Cesgranrio - BNB (Adaptada 2024)

Ao estabelecer uma conexão JDBC com o SQLite, a captura de exceções é mandatória. Em Java, SQLException é classificada como uma exceção do tipo Checked Exception. Isso implica que:

  1. O compilador não obriga o seu tratamento, cabendo ao desenvolvedor tratá-la em tempo de execução.
  2. Herda diretamente de RuntimeException.
  3. O compilador exige obrigatoriamente que ela seja tratada com um bloco try-catch ou propagada via throws.
  4. Ocorre exclusivamente por erros de sintaxe SQL, nunca por falha de infraestrutura de rede.
  5. Ignora silenciosamente falhas de integridade referencial no banco de dados.

Gabarito: C

Armadilha Semântica: A banca testa o conhecimento basilar da hierarquia de Exceções em Java. Checked Exceptions (checadas pelo compilador) exigem tratamento prévio, caso contrário, o código sequer compila. Unchecked Exceptions (herdam de RuntimeException) não exigem tratamento obrigatório em tempo de compilação.
