Tipagem e Fundamentos

O Java é estático e rigoroso. Atribuir decimais a inteiros (double em int) gera erro de compilação para evitar perda de dados. A coerção requer um Casting explícito. Além disso, strings são objetos (Classes) e não tipos primitivos.

Encapsulamento (POO)

Atributos public quebram a integridade do estado. A blindagem com private é inegociável, forçando o acesso via métodos (Getters/Setters) onde regras de negócio são aplicadas.

Lógica Preventiva

Um bom software previne falhas antes que ocorram. Erros lógicos como divisão por zero devem ser bloqueados antecipadamente com estruturas condicionais (if/else). Além disso, não se esqueça: o Construtor nasce junto com o objeto (via comando new), garantindo seu estado inicial.

Persistência (JDBC)

O padrão DAO centraliza e isola o código de banco. O Java puro não compreende as instruções do SQLite nativamente, precisando do Driver JDBC para atuar como o embaixador e tradutor oficial das operações de banco de dados.
