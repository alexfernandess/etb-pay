Momento 01

A Base do Cofre

Analisando nossa classe Conta.java do projeto, temos atributos como numero, saldo e o titular (Cliente). Vamos consolidar um único método responsável pelas transferências PIX.

Conta.java 

public class Conta {
    protected int numero;
    protected double saldo;
    protected Cliente cliente;

    public Conta(int numero, Cliente cliente) {
        this.numero = numero;
        this.cliente = cliente;
        this.saldo = 0.0;
    }

    // Método Único Original
    public boolean realizarOperacaoPIX(String chavePix, double valor) {
        System.out.println("Enviando PIX para a chave: " + chavePix);
        System.out.println("Valor da Transferência: R$ " + valor);
        return true;
    }
}
Momento 02

Sobrecarga (Alterando Parâmetros)

A Sobrecarga (Overload) acontece dentro da MESMA classe. Nós usamos exatamente o mesmo nome de método, mas alteramos os parâmetros para que ele se comporte de maneiras diferentes.

Regra do Java

O Java não diferencia os métodos pelo nome da variável. Se você criar (String cpf, double valor) e (String email, double valor), vai dar erro! Para resolver isso na Sobrecarga, precisamos mudar a quantidade ou o tipo dos parâmetros.

Conta.java (Métodos Lado a Lado) 

public class Conta {
    
    // 1. O Método Original
    public boolean realizarOperacaoPIX(String chavePix, double valor) {
        System.out.println("PIX via Chave: " + chavePix);
        return true;
    }

    // 2. SOBRECARGA: Mudando a quantidade de parâmetros
    public boolean realizarOperacaoPIX(String chavePix, double valor, boolean enviaRecibo) {
        System.out.println("PIX via Chave: " + chavePix);
        if (enviaRecibo) {
            System.out.println("Recibo em PDF enviado para o cliente!");
        }
        return true;
    }

    // 3. SOBRECARGA: Mudando o tipo de parâmetro (POO Pura)
    public boolean realizarOperacaoPIX(Conta contaDestino, double valor) {
        System.out.println("PIX Direto para a Conta Número: " + contaDestino.getNumero());
        return true;
    }
}
Momento 03

Sobrescrita (Alterando a Regra)

A Sobrescrita (Override) acontece quando criamos uma classe Filha. A nossa sugestão aqui é criar uma ContaEmpresarial que herda de Conta. Ela usa a mesma assinatura do método original, mas sobrescreve a regra para cobrar uma taxa de serviço.

@Override

O @Override é uma anotação que diz ao Java: "Estou ignorando o método do meu pai e criando o meu próprio comportamento, mas mantendo a exata mesma estrutura".

ContaEmpresarial.java 

// ContaEmpresarial herda de Conta (Extends)
public class ContaEmpresarial extends Conta {
    
    // Taxa específica para contas PJ
    private static final double TAXA_SERVICO = 1.50;

    public ContaEmpresarial(int numero, Cliente cliente) {
        super(numero, cliente);
    }

    // SUGESTÃO DE SOBRESCRITA: Mudando o comportamento do Pai
    @Override
    public boolean realizarOperacaoPIX(String chavePix, double valor) {
        
        double valorTotalDesconto = valor + TAXA_SERVICO;
        
        System.out.println("[CONTA PJ] Iniciando transferência com tarifa...");
        System.out.println("Valor enviado: R$ " + valor);
        System.out.println("Custo total debitado: R$ " + valorTotalDesconto);
        
        return true; 
    }
}
