Fundamentos: Tipos de Dados

O Java é uma linguagem fortemente tipada. Antes de construirmos a Calculadora, compreenda os 8 tipos primitivos do Java divididos em suas famílias lógicas:

Família dos Inteiros (Numéricos Puros)

Guardam números sem casas decimais. O padrão do Java é o int (32 bits), ideal para contar opções e IDs da Calculadora. Seus irmãos de memória são o byte (8 bits), short (16 bits) e o gigante long (64 bits, usado para grandes volumes de dados ou timestamps).

int opcaoMenu = 1;
long idTransacao = 9999999999L;

Família dos Decimais (Ponto Flutuante)

Armazenam números reais com frações decimais. O padrão do Java é o double (64 bits), que oferece alta precisão para cálculos. Seu irmão mais leve é o float (32 bits). CUIDADO: O Java enxerga qualquer decimal literal como double; para forçar um float, você deve usar obrigatoriamente o sufixo 'f' no final do número!

double resultadoPreciso = 15.50;
float valorSimples = 3.14f;

Família dos Lógicos (Os Interruptores)

Representada exclusivamente pelo tipo primitivo boolean. Ele ocupa apenas 1 bit de informação na JVM e é capaz de armazenar estritamente dois estados: 'true' (verdadeiro) ou 'false' (falso). É o motor fundamental dos nossos laços de repetição baseados em condições.

boolean calculadoraLigada = true;

Texto vs Caractere (A Fronteira do Objeto)

O tipo primitivo char guarda um único caractere ou símbolo Unicode e deve ser isolado usando aspas simples (ex: 'S'). Já o tipo String (com S maiúsculo) NÃO é um tipo primitivo! É uma Classe de referência que manipula cadeias de texto completas encapsuladas em aspas duplas.

char operacao = '+';
String mensagemErro = "Divisão por Zero";

O Cérebro da Máquina: Fluxos de Controle

Um programa em Java executa as linhas de cima para baixo. Mas para criar "inteligência" (tomar decisões ou repetir tarefas), precisamos dos Comandos de Controle.

Decisões ( IF / ELSE )

Verifica uma condição. Se for verdadeira, entra no bloco. Ideal para validar se o usuário tentou dividir por zero.

if (valor2 == 0) {
    System.out.println("Erro matemático");
} else {
    double result = valor1 / valor2;
}

Múltiplas Escolhas ( SWITCH )

Quando você tem várias opções (como um Menu: 1-Somar, 2-Subtrair), o switch é muito mais limpo do que escrever dezenas de if/else.

switch (opcao) {
    case 1:
        somar();
        break; // Para o código não invadir o case 2
    case 2:
        subtrair();
        break;
}

Repetição ( WHILE )

Executa um bloco de código repetidas vezes enquanto uma condição for verdadeira. Na nossa calculadora, usaremos para manter o programa aberto após cada cálculo.

while (ligada == true) {
    mostrarMenuCalculadora();
    // Se o usuario digitar sair, mudamos ligada para false
}

Tática de Imersão (Radar da Banca)

Revisaremos conceitos cruciais de POO. Entenda o que as bancas cobram para poder aplicar o conhecimento na construção da nossa Calculadora.

Classe e Objeto

A classe é a "planta baixa" que define como o sistema será. O Objeto é a "casa construída" na memória pelo comando new.

Encapsulamento (private)

O modificador private protege seus dados. A banca tentará afirmar que itens privados são acessados fora da classe, o que é falso. Eles ficam "presos" dentro dela (exigindo getters/setters).

Herança e Polimorfismo

Herança (extends) reaproveita código. Polimorfismo permite executar comandos sem saber a classe filha específica. ("Aberto para extensão, fechado para modificação").

Deploy Web (WAR)

Enquanto o .JAR embala bibliotecas normais (desktop), o .WAR (Web Application Archive) embala seu projeto para servidores web.

Passo 4 - Quadrix (Encapsulamento e IF)

Na nossa calculadora, as variáveis foram declaradas com o modificador private para garantir a proteção. A categoria de visibilidade "private" garante que esses membros somente podem ser acessados fora da classe em que foram declarados.

Gabarito: ERRADO

Membros private somente podem ser acessados DENTRO da própria classe onde foram declarados. O encapsulamento permite que a gente valide as entradas!

CalculadoraBasica.java

Ação: Altere as suas variáveis para private. Crie o método de divisão e utilize a condicional IF para impedir a temida divisão por zero!

private double valor1;
private double valor2;

// Usando controle de fluxo IF para proteger o cálculo!
public double dividir(double v1, double v2) {
    if (v2 == 0) {
        System.out.println("Erro: Divisão por Zero!");
        return 0; // Encerra o fluxo
    }
    return v1 / v2;
}

Passo 6 - Quadrix (POO / Herança)

Para poupar código, utilizamos Herança. Por meio desse relacionamento interclasses, é permitido que a subclasse CalculadoraAvancada herde de forma automática os atributos e métodos (como a divisão segura) de sua superclasse CalculadoraBasica.

Gabarito: CERTO

Passo 8 - CEBRASPE (Pegadinha do Float Literal)

No desenvolvimento da nossa Calculadora Básica em Java, ao tentarmos inicializar uma variável de precisão decimal simples escrevendo diretamente a linha de instrução float resultado = 5.5;, o compilador do Java executará o código com sucesso, pois reconhece automaticamente o número fracionário como um tipo float compatível.

Gabarito: ERRADO

Pega clássico de prova! Por padrão, todo literal decimal no Java é tratado como 'double'. Atribuir um double a um float gera erro de compilação por perda de precisão (loss of precision). O correto seria usar '5.5f'.

AppCalculadora.java

Ação: Abra seu código e teste: declare um float sem o 'f' no final e veja o compilador travar. Corrija adicionando o sufixo 'f' para blindar a tipagem.

// ERRO: float resultado = 5.5; (compilador vê 5.5 como double)
float resultado = 5.5f; // Correto usando o sufixo 'f'
Passo 9 - Quadrix (String como Tipo Primitivo)

O ecossistema Java 21 é fortemente tipado e divide suas estruturas de dados entre primitivos e referências. O tipo 'String', amplamente utilizado no nosso projeto para exibir mensagens de erro customizadas no terminal da calculadora, é classificado nativamente como um dos 8 tipos primitivos da linguagem.

Gabarito: ERRADO

O Java possui apenas 8 tipos primitivos (byte, short, int, long, float, double, char, boolean). A 'String' é uma Classe (tipo de objeto/referência) e por isso começa com letra maiúscula.

AppCalculadora.java

Ação: Adicione uma String de log no seu método principal para registrar as ações do usuário, diferenciando-a claramente das variáveis numéricas primitivas.

double numeroPrimitivo = 10.5; // Tipo primitivo (double)
String logMensagem = "Log: Calculando..."; // Tipo de referência (Classe String)
Passo 10 - Quadrix (Convenções de Nomenclatura)

De acordo com as convenções oficiais da linguagem Java adotadas para manter o Clean Code, os nomes de classes e interfaces devem utilizar a convenção camelCase (ex: calculadoraBasica), ao passo que variáveis de instância e nomes de métodos devem utilizar estritamente a convenção PascalCase (ex: PrimeiroNumero).

Gabarito: ERRADO

A banca inverteu as regras. Classes usam PascalCase (Iniciais Maiúsculas, ex: CalculadoraBasica). Variáveis e métodos usam camelCase (primeira letra minúscula, ex: primeiroNumero, calcularSoma).

Git e Fluxos de Controle (Fundamentos)

  • Git: git init (inicia), git add . (prepara), git commit -m (salva histórico).
  • IF / ELSE: Comando de decisão. Valida uma condição lógica antes de executar um bloco (ex: impedir divisão por zero).

  • WHILE: Laço de repetição. Executa código continuadamente enquanto a condição for true.

Q1 - POO / Extensão e Polimorfismo

  • O que a questão disse: Ao introduzirmos Polimorfismo, para calcular diferentes operações, o sistema básico teria que sofrer profundas modificações em cada chamada de classe. (ERRADO)
  • O Erro (O Quê): Afirmar que a arquitetura polimórfica exige modificações profundas no sistema raiz para adicionar ou estender comportamentos.
  • A Regra (Por Quê): O polimorfismo e o binding dinâmico permitem estender comportamentos através de interfaces comuns (ex: Operacao) sem necessidade de modificar a classe-raiz.

Q2 - Java Web / RESTful JSON

  • O que a questão disse: Em arquitetura RESTful, o resultado da operação matemática deve ser devolvido obrigatoriamente e de forma exclusiva no formato XML. (ERRADO)
  • O Erro (O Quê): Restringir o formato de retorno de uma API RESTful de forma obrigatória e exclusiva ao XML.
  • A Regra (Por Quê): APIs RESTful suportam múltiplos formatos de retorno (JSON, XML, HTML, etc.), sendo o JSON o padrão mais leve e utilizado do mercado moderno.

Q3 - Java Web / Deploy

  • O que a questão disse: A extensão padronizada no Java para empacotamento web é .WAR. (CERTO)
  • O Erro (O Quê): N/A (A afirmação é verdadeira).
  • A Regra (Por Quê): Arquivos .WAR (Web Application Archive) empacotam componentes da camada web (servlets, JSPs, HTML). Arquivos .JAR (Java Archive) empacotam aplicações standalone ou bibliotecas.

Q4 - POO / Encapsulamento

  • O que a questão disse: Membros private só podem ser acessados fora da classe em que foram declarados. (ERRADO)
  • O Erro (O Quê): Afirmar que membros de acesso private são acessíveis fora de sua classe declarada.
  • A Regra (Por Quê): O modificador de acesso private restringe a visibilidade do membro estritamente a dentro da própria classe. Acessos externos devem ser controlados via Getters/Setters.

Q5 - POO / Conceito de Classe

  • A Regra (Por Quê): A classe funciona como um molde ou planta baixa estrutural que define as variáveis de instância (atributos) e métodos (comportamentos) dos objetos.

Q6 - POO / Herança
