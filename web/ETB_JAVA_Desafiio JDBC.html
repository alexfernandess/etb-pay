Desafio: Motor de Contas ETB-Pay

Bem-vindos à fase técnica do processo seletivo do ETB-Pay. O objetivo central é criar, do zero, o projeto Java chamado desafioJDBC para servir como motor de persistência de clientes e contas da nossa Fintech utilizando Java Puro e JDBC. O projeto deve ser estruturado obrigatoriamente seguindo o padrão arquitetural DAO (Data Access Object) e conter uma classe dedicada chamada ConexaoDB para gerenciar o driver de conexão. O desafio elimina a complexidade de frameworks web (como Spring Boot) para focar estritamente no domínio de Orientação a Objetos (Associação) e persistência no banco SQLite.

Atenção Arquitetural

Proibido vincular uma conta no exato momento da criação do cliente. O fluxo exige que o cliente seja criado primeiro. A conta será vinculada posteriormente utilizando a Chave Estrangeira (ID do Cliente).

1. Modelagem do Banco (SQLite)

Utilizaremos o SQLite para evitar problemas de rede no laboratório. O desenvolvimento do projeto será realizado de forma FASEADA. Nesta PRIMEIRA ETAPA, o foco exclusivo da squad é criar a base de dados SQLite diretamente pelo VS Code (utilizando suas extensões de banco de dados) e codificar as classes de negócio (entidades): Cliente.java e Conta.java. Execute o script SQL abaixo na sua IDE para criar a estrutura inicial. 

-- Tabela Pai: Clientes
CREATE TABLE clientes (
    id_cliente INTEGER PRIMARY KEY AUTOINCREMENT,
    cpf VARCHAR(11) NOT NULL UNIQUE,
    nome VARCHAR(100) NOT NULL,
    email VARCHAR(100) NOT NULL,
    telefone VARCHAR(15)
);

-- Tabela Filha: Contas (Relacionamento 1:N)
CREATE TABLE contas (
    id_conta INTEGER PRIMARY KEY AUTOINCREMENT,
    numero_conta VARCHAR(20) NOT NULL UNIQUE,
    tipo_conta VARCHAR(20) NOT NULL, -- 'CORRENTE' ou 'POUPANCA'
    saldo DECIMAL(10, 2) NOT NULL DEFAULT 0.00,
    id_cliente INTEGER NOT NULL,
    FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente)
);

2. Requisitos do Sistema (Menu Console)

Sua aplicação Java deve ter um loop while com um switch-case contendo as 4 opções abaixo. Use a classe Scanner para capturar os dados.

1

Cadastrar Novo Cliente

Solicite CPF, Nome, Email e Telefone. Realize um INSERT na tabela clientes. Não insira nenhuma conta neste momento.

2

Listar Base de Clientes

Execute um SELECT * FROM clientes e imprima os dados no console de forma formatada.

3

Vincular Conta a um Cliente

Core Business do Desafio: Solicite o ID do Cliente. Valide se ele existe no banco. Se existir, peça o Número da Conta e o Tipo (Corrente/Poupança). Insira na tabela contas vinculando ao ID informado.

> Informe o ID do Cliente: 1
> [Sucesso] Cliente 'Alex' encontrado.
> Informe o número da nova conta: 10234-5
> Conta vinculada com sucesso!
4

Buscar Contas por Cliente

Solicite o ID do Cliente e execute um SELECT * FROM contas WHERE id_cliente = ? para exibir o extrato de contas atreladas a ele.

Critérios de Avaliação

🚀 Atenção Squad 3G - Entrega Faseada: Apenas o item Nº 1 (Modelagem de Classes) faz parte do entregável obrigatório desta primeira etapa (Criação do banco de dados SQLite e das classes de negócio Cliente e Conta). Os itens Nº 2 e Nº 3 serão integralmente desenvolvidos e codificados, de forma coletiva em sala de aula.

Criação correta das classes Cliente e Conta, aplicando os conceitos de associação.

2. Persistência (Unidades 9 e 10)

Uso correto de PreparedStatement para evitar SQL Injection nas rotinas de banco.

3. Tratamento de Exceções

Uso de blocos try-catch para lidar com falhas de conexão no DriverManager.

Setup do Projeto

  • JDK 17 ou superior instalado (opcional).
  • Instalar a extensão Extension Pack for Java (da Microsoft) no VS Code para habilitar o suporte completo à compilação e execução automatizada do projeto.
  • Driver JDBC do SQLite referenciado no projeto (sqlite-jdbc.jar).
  • Estrutura obrigatória seguindo o padrão arquitetural DAO (Data Access Object) e contendo a classe dedicada ConexaoDB para gerenciar a conexão.

Dica Tech

String de conexão para o SQLite:
jdbc:sqlite:etbpay.db

O arquivo do banco será criado automaticamente na raiz do seu projeto ao executar a primeira conexão.
