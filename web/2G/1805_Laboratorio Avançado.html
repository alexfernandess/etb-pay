Constraints: A Linha de Defesa Final

A integridade real não ocorre no Front-end ou Back-end, mas no disco físico. As Constraints são guardrails irremovíveis de mercado corporativo.

NOT NULL & UNIQUE

O NOT NULL impede a inserção nula de dados sensíveis. O UNIQUE cria uma trava absoluta no disco contra a duplicidade de chaves, como múltiplos cadastros com um único CPF ou E-mail.

CHECK Constraint

Atua como uma barreira algébrica e lógica nativa da engine. A instrução CHECK avalia expressões matemáticas ou strings antes de autorizar qualquer gravação de I/O física.

A Estratégia Corporativa (Shadow Table)

O SQLite impõe limitações em modificar tabelas ativas em produção. Para resolver, utilizamos o padrão profissional de Tabela Espelho (Shadow Table):

-- 1. Isolar a tabela legada mantendo os dados intactos
ALTER TABLE clientes RENAME TO clientes_old;

-- 2. Engenharia do novo esquema blindado (Prefixo 'tb_')
CREATE TABLE tb_clientes (
  id_cliente INTEGER PRIMARY KEY AUTOINCREMENT,
  nome_completo TEXT NOT NULL,
  cpf TEXT UNIQUE NOT NULL,
  email TEXT UNIQUE NOT NULL,
  CHECK (length(cpf) = 14)
);

Migração Condicional e Saneamento

A estrutura `tb_clientes` está pronta. Precisamos reaver a massa de dados extraída da rede, ignorando ativamente os registros anômalos.

-- DML: Pipeline de migração com filtragem cirúrgica
INSERT INTO tb_clientes (cpf, nome_completo, email)
SELECT cpf, nome, email
FROM clientes_old
WHERE length(cpf) = 14; -- Apenas CPFs formatados passam

Proteção Atômica via Shadow Table

A adoção do padrão de Shadow Table permitiu a higienização da massa de dados corrompida e a inserção de guardrails físicos na tabela sem downtime destrutivo.

Pipeline de Comandos Técnicos Consolidado:

-- 1. Isolamento Corporativo (Tabela Legada)
ALTER TABLE clientes RENAME TO clientes_old;

-- 2. Engenharia de Schema Blindado (Constraints)
CREATE TABLE tb_clientes (
    id_cliente INTEGER PRIMARY KEY AUTOINCREMENT,
    nome_completo TEXT NOT NULL,
    cpf TEXT UNIQUE NOT NULL,
    email TEXT UNIQUE NOT NULL,
    CHECK (length(cpf) = 14)
);

-- 3. Migração Cirúrgica com Higienização (Filtro)
INSERT INTO tb_clientes (cpf, nome_completo, email)
SELECT cpf, nome, email FROM clientes_old 
WHERE length(cpf) = 14;
