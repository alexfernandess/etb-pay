01. Marco Zero - Reconstrução e Estrutura

Após a corrupção do banco original, nossa Squad assume a reconstrução imediata do ecossistema agencia_bancaria.db. O foco é a padronização e a integridade absoluta para o hand-off de amanhã com a equipe de Java.

📚 O que é DDL?

A DDL (Data Definition Language) define a estrutura "física" onde os dados moram. Seus comandos principais são o CREATE, ALTER e DROP.

Construindo o Cofre (DDL)

-- Criação das tabelas centrais usando o padrão snake_case
CREATE TABLE clientes (
    id_cliente INTEGER PRIMARY KEY AUTOINCREMENT,
    nome_completo TEXT NOT NULL,
    cpf TEXT UNIQUE NOT NULL
);

CREATE TABLE contas (
    id_conta INTEGER PRIMARY KEY AUTOINCREMENT,
    id_cliente INTEGER NOT NULL,
    saldo REAL DEFAULT 0.0,
    FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente)
);

📚 O que é DML?

A DML (Data Manipulation Language) gerencia a vida dentro das tabelas criadas. Seus comandos essenciais são INSERT, UPDATE e DELETE.

02. O Motor Financeiro - Lógica e Precisão

O Banco de Dados não é apenas um repositório inerte; ele é o verdadeiro motor de cálculo da Fintech. Aqui, validamos as rotinas de busca e as transações matemáticas que a Squad de Java irá consumir.

Auditoria de Precisão (DQL / WHERE)

Nós usamos a cláusula WHERE para atuar como um filtro, impedindo o sobrecarregamento de memória do servidor de aplicação.

-- 1. Busca cirúrgica por CPF (O Java precisa dessa query)
SELECT * FROM clientes WHERE cpf = '111.111.111-11';

-- 2. Identificar clientes VIP (Saldo superior a R$ 4.000)
SELECT id_cliente, saldo FROM contas WHERE saldo > 4000.0;

A Engenharia do PIX (UPDATE Matemático)

⚠️ A Regra de Ouro da Arquitetura

A matemática financeira deve ocorrer nativamente dentro do Motor do Banco de Dados para garantir consistência. Nunca fazemos cálculos na memória do Java e enviamos o resultado seco.

-- Cenário 1: Transferência (Alex envia R$ 500 para Lillianne)
UPDATE contas SET saldo = saldo - 500.0 WHERE id_conta = 1;
UPDATE contas SET saldo = saldo + 500.0 WHERE id_conta = 2;

-- Cenário 2: Bônus Rendimento (Aumentar 10% para contas VIPs)
UPDATE contas SET saldo = saldo * 1.1 WHERE saldo > 5000.0;

03. Compliance e Hand-off

Integridade referencial e o cumprimento da Lei Geral de Proteção de Dados (LGPD) são a base para a homologação final do nosso ambiente ETB-PAY.

O Perigo do DELETE e a Faxina Hierárquica

Tentar apagar um cliente que possui uma conta ativa resulta em um erro de restrição do banco. Isso não é um bug, é Segurança por Design.

-- Tentativa de exclusão direta (Resulta em erro de Foreign Key)
DELETE FROM clientes WHERE id_cliente = 1;

-- Ordem de encerramento correta (Aprovada pelo Compliance):
-- 1º Limpamos os vínculos da conta
DELETE FROM contas WHERE id_cliente = 1;
-- 2º Excluímos os dados biográficos da pessoa física
DELETE FROM clientes WHERE id_cliente = 1;

🛡️ Propriedades ACID em Transações

Nossa arquitetura garante que cada operação seja: Atômica (Tudo ou nada), Consistente (Respeita o esquema), Isolada (Sem interferências em concorrência) e Durável (Gravação permanente no disco físico).
