01. Setup da Infraestrutura

Para garantir que toda a Squad utilize a mesma versão do motor e do projeto, acesse a pasta da rede e copie os arquivos para seu ambiente de trabalho local.

Diretório na Rede:
\\nsi12\Alex Alves\Banco de Dados

Certifique-se de copiar o executável sqlite3.exe e o arquivo agencia_bancaria.db.

02. Execução no Ambiente Local

Como um DBA Raiz, utilize o terminal para abrir o arquivo de banco de dados diretamente.

./sqlite3.exe agencia_bancaria.db

03. Consulta das Tabelas

Verifique quais tabelas já existem na estrutura física do banco.

sqlite> .tables
sqlite> .schema clientes

04. Criação das Tabelas (DDL)

Implemente a fundação do ETB-PAY seguindo o padrão snake_case para integração com Java.

CREATE TABLE clientes ( id_cliente INTEGER PRIMARY KEY AUTOINCREMENT, nome_completo TEXT NOT NULL, cpf TEXT UNIQUE NOT NULL ); CREATE TABLE contas ( id_conta INTEGER PRIMARY KEY AUTOINCREMENT, id_cliente INTEGER NOT NULL, saldo REAL DEFAULT 0.0, FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente) );

05. Inserção de Dados Fictícios

Povoamento da base com os quatro registros de referência da Squad.

INSERT INTO clientes (nome_completo, cpf) VALUES ('Ada Lovelace', '101.202.303-44'), ('Alan Turing', '505.606.707-88'), ('Margaret Hamilton', '909.808.707-11'), ('Grace Hopper', '111.222.333-44'); INSERT INTO contas (id_cliente, saldo) VALUES (1, 15000.00), (2, 8000.00), (3, 12000.50), (4, 9500.25);

06. Operações PIX entre Contas

Matemática de banco nativa: débitos e créditos simultâneos para manter a integridade.

-- PIX de R$ 500: Ada Lovelace enviando para Alan Turing UPDATE contas SET saldo = saldo - 500.0 WHERE id_conta = 1; UPDATE contas SET saldo = saldo + 500.0 WHERE id_conta = 2;
