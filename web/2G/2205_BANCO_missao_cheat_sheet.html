Missão: O "Cheat Sheet" do DBA Sênior

Fala, Squad!

Um verdadeiro Engenheiro de Dados não precisa decorar todas as sintaxes de cabeça, mas ele sabe exatamente onde consultar a documentação correta. Para consolidar a nossa operação de Saneamento de Dados, o desafio de vocês será construir o seu próprio Guia de Bolso (Cheat Sheet) de Comandos SQL.

Este documento será o seu "canivete suíço". Ele servirá como base oficial e guia de consulta para que vocês tenham autonomia total na execução dos comandos SQL nos próximos laboratórios práticos e projetos da nossa aplicação.

O que deve conter no Guia?

Seu guia deve explicar o propósito e demonstrar a sintaxe funcional dos seguintes tópicos:

  • SELECT & WHERE: Como auditar e filtrar dados corrompidos.
  • INSERT & UPDATE: Injeção e cirurgia em registros legados.
  • CREATE TABLE: Estruturação com PRIMARY KEY e AUTOINCREMENT.
  • Constraints: Uso de CHECK e UNIQUE para blindar o banco.
  • REPLACE(): O triturador de impurezas em tempo de execução.
  • GLOB: O leitor biométrico para validar padrões rigorosos.

Avaliação & Entrega

Formatos de Entrega Aceitos:

Página Web (HTML/CSS) Documento (PDF/Word) Markdown (.md)

Critérios de Aceite:

Os trabalhos serão validados rigorosamente com base em dois pilares: a precisão técnica dos exemplos de código fornecidos e a qualidade e clareza da informação gerada (explique para você mesmo do futuro como o código funciona).
