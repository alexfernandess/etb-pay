Desafio: Motor de Contas ETB-Pay

Bem-vindos à fase técnica do processo seletivo. Para otimizar o tempo, será disponibilizado um projeto base (Boilerplate) do motor de contas com a estrutura inicial pré-montada. Porém, este código legado contém falhas intencionais de arquitetura e lógica. Sua missão é baixar o projeto, configurar o ambiente e atuar como um engenheiro de software para investigar, debugar e fazer a aplicação rodar perfeitamente.

Atenção Arquitetural

Proibido vincular uma conta no exato momento da criação do cliente. O fluxo exige que o cliente seja criado primeiro. A conta será vinculada posteriormente utilizando a Chave Estrangeira (ID do Cliente).

1. Modelagem do Banco (SQLite)

O banco de dados do nosso motor de contas já está estruturado previamente seguindo as regras da Terceira Forma Normal (3FN). O script SQL (clientes.sql) já está incluso dentro do Boilerplate. Basta executá-lo diretamente no VS Code utilizando a sua extensão de banco de dados para gerar automaticamente o arquivo etbpay.db. 

-- Tabela Pai: Clientes
CREATE TABLE clientes (
    id_cliente INTEGER PRIMARY KEY AUTOINCREMENT,
    cpf VARCHAR(11) NOT NULL UNIQUE,
    nome VARCHAR(100) NOT NULL,
    email VARCHAR(100) NOT NULL,
    telefone VARCHAR(15)
);

-- Tabela Filha: Contas (Relacionamento 1:N)
CREATE TABLE contas (
    id_conta INTEGER PRIMARY KEY AUTOINCREMENT,
    numero_conta VARCHAR(20) NOT NULL UNIQUE,
    tipo_conta VARCHAR(20) NOT NULL, -- 'CORRENTE' ou 'POUPANCA'
    saldo DECIMAL(10, 2) NOT NULL DEFAULT 0.00,
    id_cliente INTEGER NOT NULL,
    FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente)
);

2. Pontos de Investigação (Missão de Troubleshooting)

Bug 1: Falha na String de Conexão

O arquivo ConexaoDB.java está apontando para o banco errado ou com a sintaxe JDBC incorreta. Encontre a string e corrija para jdbc:sqlite:etbpay.db.

Bug 2: Violação de Encapsulamento

A entidade Conta.java está com vulnerabilidades de segurança (atributos públicos ou falta de Getters/Setters). Refatore aplicando o encapsulamento correto.

Bug 3: O Mistério do INSERT fantasma

O método de salvar no ClienteDAO.java roda sem dar erro, mas o dado não vai para o banco. Verifique se o comando stmt.executeUpdate() foi chamado ou se as variáveis estão trocadas.

Setup do Projeto

  • JDK 17 ou superior instalado (opcional).
  • Instalar a extensão Extension Pack for Java (da Microsoft) no VS Code para habilitar o suporte completo à compilação e execução automatizada do projeto.
  • Driver JDBC do SQLite referenciado no projeto (sqlite-jdbc.jar).
  • Estrutura obrigatória seguindo o padrão arquitetural DAO (Data Access Object) e contendo a classe dedicada ConexaoDB para gerenciar a conexão.

Dica Tech

String de conexão para o SQLite:
jdbc:sqlite:etbpay.db

O arquivo do banco será criado automaticamente na raiz do seu projeto ao executar a primeira conexão.
