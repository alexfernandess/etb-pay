A Queda e a Planilha de Contingência

Atenção, Squad! O sistema principal do ETB-PAY sofreu uma queda massiva no servidor. Para não paralisar a operação e garantir que as transferências financeiras continuassem, nossa equipe de Suporte ativou o Plano de Contingência Manual. Eles começaram a registrar todos os fluxos de Pix utilizando uma planilha básica do Excel. A intenção foi salvar a operação, mas o resultado técnico gerou o caos da "Planilha Universal".

Alerta de Compliance Urgente

Auditoria do BACEN: Reprovação

O Banco Central do Brasil (BACEN) auditou essa planilha de contingência e a reprovou imediatamente. O laudo aponta quebra severa de compliance, redundância de dados injustificável e ausência total de integridade relacional.

A Bomba-Relógio

A nossa missão emergencial é sanear essa base caótica. Precisamos projetar uma nova arquitetura aplicando as regras de Normalização Relacional antes de pensar em importar qualquer dado para o SGBD definitivo da ETB-PAY.

Roteiro de Resgate

Antes de colocarmos a mão no banco de dados para resolver esse desastre e iniciar o nosso Laboratório Prático de Deploy, a Squad inteira precisa passar por um treinamento teórico de nivelamento. O roteiro focará nos seguintes passos:

01

Triagem e 1ª Forma Normal (1FN)

Analisaremos as anomalias do arquivo CSV fornecido. A missão é garantir a atomicidade. Vocês deverão quebrar o endereço em colunas distintas (Logradouro, Bairro, UF) e discutir qual a melhor estratégia para separar os telefones multivalorados.

02

Modelagem Relacional (2FN e 3FN)

A planilha gasta espaço repetindo as informações bancárias e de endereço a cada Pix realizado. Vocês deverão projetar o modelo conceitual extraindo os dados para novas entidades (tb_clientes, tb_contas, tb_transacoes) estabelecendo as Chaves Primárias (PK) e Estrangeiras (FK).

03

Deploy: DDL e Carga de Dados

Com o Diagrama validado, abriremos o SGBD. A Squad irá codificar os comandos CREATE TABLE estruturais e gerar os scripts de INSERT INTO para transferir o log manual para o ecossistema protegido do banco.

ETB-PAY: Cheat Sheet de SQL (DDL e DML)

Seção 1: Criação de Tabelas (DDL)

CREATE TABLE tb_clientes (
    id_cliente INT PRIMARY KEY,
    nome_completo VARCHAR(100) NOT NULL
);

CREATE TABLE tb_contas (
    id_conta INT PRIMARY KEY,
    id_cliente INT,
    agencia VARCHAR(10),
    numero_conta VARCHAR(20),
    FOREIGN KEY (id_cliente) REFERENCES tb_clientes(id_cliente)
);

Seção 2: Inserção de Dados (DML)

INSERT INTO tb_clientes (id_cliente, nome_completo) 
VALUES (1, 'João Silva');

INSERT INTO tb_contas (id_conta, id_cliente, agencia, numero_conta) 
VALUES (10, 1, '0001', '12345-6');

Seção 3: Consultas (DQL)

SELECT 
    c.nome_completo,
    co.agencia,
    co.numero_conta
FROM tb_clientes c
JOIN tb_contas co ON c.id_cliente = co.id_cliente;
